# GoMad 公开路线图

GoMad 是上游敏捷 AI 驱动开发框架的硬分叉。v1.1 里程碑聚焦于重命名、瘦身、署名与品牌；
上游已有功能之外的新 Agent 与 Skill 将推迟到后续里程碑处理。

<div class="roadmap-container">

  <h2 class="roadmap-section-title">里程碑 1 (v1.1.0) - 进行中</h2>

  <div class="roadmap-future">
    <div class="roadmap-future-card">
      <h4>原子化重命名</h4>
      <p>完成文件系统与文本级别的全量重命名，确保发布包中不再泄漏任何上游命名。</p>
    </div>
    <div class="roadmap-future-card">
      <h4>署名与合规</h4>
      <p>逐字节保留上游 MIT 许可，依据合理使用原则改写商标说明，并在 README 中清晰致谢原作者。</p>
    </div>
    <div class="roadmap-future-card">
      <h4>瘦身</h4>
      <p>移除 GoMad 不再承载的外部模块清单、builder 与 web bundle 相关残留。</p>
    </div>
    <div class="roadmap-future-card">
      <h4>品牌</h4>
      <p>更新 banner、wordmark、favicon 与 CLI 启动画面，使视觉层与命名层对齐。</p>
    </div>
  </div>

  <h2 class="roadmap-section-title">里程碑 2 及之后</h2>

  <div class="roadmap-future">
    <div class="roadmap-future-card">
      <h4>GoMad 专属 Agent</h4>
      <p>面向 GoMad 使用场景的新 Agent，扩展核心敏捷工作流。</p>
    </div>
    <div class="roadmap-future-card">
      <h4>GoMad 专属 Skill</h4>
      <p>面向 GoMad 使用者新建的 Skill，而非整体照搬上游。</p>
    </div>
  </div>
