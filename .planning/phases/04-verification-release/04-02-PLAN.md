  <action>
Create `test/test-e2e-install.js` following the exact test pattern from `test/test-installation-components.js` (CommonJS, assert helper, colors, passed/failed counters).

**Script structure:**

```
1. Pack: execSync('npm pack') -- captures tarball filename from stdout
2. Create temp dir: fs.mkdtempSync(path.join(os.tmpdir(), 'gomad-e2e-'))
3. Init: execSync('npm init -y', { cwd: tempDir })
4. Install: execSync(`npm install ${tarballPath}`, { cwd: tempDir })
5. Verify structure:
   - Assert tempDir/node_modules/@xgent-ai/gomad/src/ exists
   - Assert tempDir/node_modules/@xgent-ai/gomad/tools/installer/ exists
6. Verify skills (D-07):
   - Glob for all gm-* directories under tempDir/node_modules/@xgent-ai/gomad/src/
   - For each gm-* directory, assert SKILL.md exists at that path
   - Count total skills found, assert > 0
7. Cleanup: fs.removeSync(tempDir), fs.unlinkSync(tarballPath)
8. Exit: process.exit(failed > 0 ? 1 : 0)
```

Use `fs-extra` (already a dependency) for `fs.mkdtempSync`, `fs.removeSync`. Use `glob.sync` (already a dependency) for finding gm-* directories.

Also add to package.json scripts: `"test:e2e": "node test/test-e2e-install.js"`

**Important:** This test does NOT run `gomad install` interactively (per Pitfall 2 in RESEARCH.md). Instead it verifies the tarball contents structurally -- checking that all skill directories and SKILL.md files are present. This avoids the interactive @clack/prompts hang issue.

**Timeout:** Set a 120-second timeout on the npm install execSync call (`{ timeout: 120000 }`) to prevent hanging.
