<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source tree → destination tree | `fs.copy(sourceDir, skillDir)` crosses from `gomadDir` (installer-owned) to `targetPath` (user workspace). Both paths are installer-composed from `projectDir` + config-file-derived segments — NO user-runtime input. |
| symlink indirection | Pre-upgrade (v1.1) may have a symlink at `skillDir` pointing back into `gomadDir` or elsewhere. Our `fs.unlink` removes the link only, not its target. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | `fs.copy` over a symlink | mitigate | D-20 pre-check unlinks the symlink BEFORE `fs.copy` runs, so the copy writes real files into `targetPath` — it never follows a symlink back into the source tree. `fs.copy` default dereferences symlinks in source, which is the intended behavior here (we want content, not link structure). |
| T-06-02 | Information disclosure | reading destination stat | accept | `fs.lstat(skillDir)` only exposes whether the path is a symlink to the installer's own log output. Low severity — installer runs with user-level privileges and writes to user's own workspace. |
| T-06-03 | Denial of service | failed `fs.unlink` on read-only FS or permission denial | mitigate | The `await fs.unlink(skillDir)` call can throw; existing fs-extra semantics propagate the error. Node.js fs errors include an actionable message (`EPERM`, `EACCES`). No special handling needed — installer surfaces the error to the user, who can fix permissions. Phase 7 will add backup + dry-run safety; Phase 6 surfaces raw errors (acceptable per CONTEXT.md interim posture D-21). |
| T-06-04 | Elevation of privilege | path traversal via `canonicalId` | accept | `canonicalId` is sourced from `skill-manifest.csv` which is owned by the distributed package (signed on npm). User-controlled path segments do NOT reach `path.join(targetPath, canonicalId)` in this plan. Phase 7's realpath containment (INSTALL-06) layers defense-in-depth for the manifest-driven cleanup path. |

No high-severity unmitigated threats. Plan is safe to execute.
</threat_model>
