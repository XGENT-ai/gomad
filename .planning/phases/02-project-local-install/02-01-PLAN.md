<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm package -> user project | gomad copies files from its bundled assets/ into the user's ./.claude/ directory |
| user project -> filesystem | installer reads/writes within process.cwd() scope |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | tools/installer.js | mitigate | installFiles() copies from ASSETS_SRC (inside npm package) only -- no user-controlled source paths. Target is always join(process.cwd(), '.claude') -- no path traversal possible since paths are constructed from constants, not user input |
| T-02-02 | Information Disclosure | tools/installer.js | accept | Manifest file lists installed file paths but contains no secrets. Manifest lives in project directory under git control |
| T-02-03 | Denial of Service | tools/installer.js uninstall | mitigate | Uninstall reads manifest to identify files -- only removes files listed in manifest, never does recursive rmSync on .claude/ directory. Prevents accidental deletion of user files |
| T-02-04 | Elevation of Privilege | tools/installer.js | accept | CLI runs with user's own permissions. No privilege escalation vector -- writes only to cwd which user already owns |
</threat_model>
