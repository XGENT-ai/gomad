    - Idempotent: a second run on a clean tree must report `files_touched: 0` and exit 0.

    - Substitution mapping applied longest-first (see context interfaces block).
    - `bmm` substitution is anchored to word boundaries or identifier segments only — must NOT rewrite unrelated trigrams. Use a regex like `/\bbmm\b/g` or a look-around that matches `bmm` only when surrounded by non-alphanumeric characters (so `bmm-style.csv` matches, but `xbmmx` does not).
    - Script is CommonJS (matches repo convention, D-05).
    - Script must not crash on binary files; it filters by extension (target glob set) and skips anything else.
  </behavior>
  <action>
    Create `tools/dev/rename-sweep.js` (CommonJS). Implementation:

    1. Glob target files using the `glob` package already in deps (or `fs.readdirSync` recursion if glob is awkward here):
       Pattern: `**/*.{md,yaml,yml,json,js,mjs,cjs,ts,astro,html,csv}`
       Ignore: `node_modules/**`, `.git/**`, `build/**`, `.planning/**`

    2. Hard-code the exclude list as a Set of exact relative paths:
       ```js
       const EXCLUDE_FILES = new Set([
         'LICENSE',
         'CHANGELOG.md',
         'TRADEMARK.md',
         'CONTRIBUTORS.md',
         'README.md',
         'README_CN.md',
       ]);
       ```

    3. Define ordered mappings (longest-first):
       ```js
       const MAPPINGS = [
         { re: /BMAD Method/g, to: 'GoMad', name: 'BMAD Method→GoMad' },
         { re: /bmad-method/g, to: 'gomad', name: 'bmad-method→gomad' },
         { re: /BMad/g, to: 'GoMad', name: 'BMad→GoMad' },
         { re: /BMAD/g, to: 'GOMAD', name: 'BMAD→GOMAD' },
         { re: /bmad/g, to: 'gomad', name: 'bmad→gomad' },
         { re: /(?&lt;![A-Za-z0-9_])bmm(?![A-Za-z0-9_])/g, to: 'gomad', name: 'bmm→gomad (word-anchored)' },
       ];
       ```

    4. Apply mappings per-file in order; track replacement counts per mapping and per file.

    5. Support `--dry-run` flag that skips `fs.writeFileSync` but still reports would-change counts.

    6. Print summary at end:
       ```
       Files scanned: N
       Files touched: M
       Files skipped (exclude list): K
       Replacements by mapping:
         BMAD Method→GoMad: X
         ...
       ```

    7. Exit code 0 on success; non-zero on error (permission, parse failure on a non-text file, etc.).

    Test BEFORE sweeping the repo:
    - Run `node tools/dev/rename-sweep.js --dry-run` on current repo state.
    - Confirm it reports non-zero files_touched with a plausible summary.
    - Confirm it reports zero touches for files in the exclude list.

    Commit: `feat(02-02): add tools/dev/rename-sweep.js idempotent content-sweep script (D-05)`.
