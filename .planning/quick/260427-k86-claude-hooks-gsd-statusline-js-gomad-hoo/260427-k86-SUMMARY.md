## Deviations from plan

None of substance. Two minor adjustments worth noting:

1. **Removed shebang from `gomad-statusline.js`.** The plan called for `#!/usr/bin/env node`, but ESLint's `n/hashbang` rule (active for `tools/**/*.js`) flagged it because the file isn't a `bin` entry — it's an asset. Resolution: leave the shebang off. Runtime invocation is via `node "$CLAUDE_PROJECT_DIR"/.claude/hooks/gomad-statusline.js` (the registered command in `settings.json`), so the shebang was never load-bearing. The file's exec bit is still set (0755) for hygiene, and unit-testing via `node path/to/file.js < /dev/null` works identically. This diverges from the GSD reference at `.claude/hooks/gsd-statusline.js`, which has a shebang — but that path is in the eslint ignores list (`.claude/**`), our tools-tree path is not.
2. **Implementation file size 339 LOC vs the plan's "under ~200 LOC" suggestion.** The actual logic is tight, but jsdoc comments + the dedicated YAML parser (~70 LOC for `parseFlatYaml` + `parseSprintYaml`) push the total up. Trimming further would require dropping comments that document non-obvious silent-fail paths, which would hurt maintainability. Calling this acceptable.
