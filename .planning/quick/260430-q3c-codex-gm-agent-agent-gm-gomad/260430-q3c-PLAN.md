<objective>
Codex (and every other platform without `launcher_target_dir`) currently installs skill content verbatim from `src/gomad-skills/`, but the source bodies reference agents as `/gm:agent-pm` etc. Codex cannot resolve colon-form invocations — only `$gm-agent-pm` (dash, dollar-prefixed). This makes installed Codex skill content fail at runtime when the user's flow tells them to "invoke `/gm:agent-pm`".

Fix: install-time syntax transformation, gated on a new `invocation_syntax` field in `platform-codes.yaml`. Source stays canonical (colon); the transform fires when copying to a non-launcher IDE.

Purpose: Restore working agent invocation references inside installed Codex/Auggie/Cline/CodeBuddy/Antigravity skill content without forking the source.

Output: A two-line config field, a transform helper, two install-pipeline call sites, and a unit test proving the regex is conservative.
</objective>

    Conservative regex transform — these are the ONLY rewrites permitted:

    Given input string `s` and `shortNames = ['analyst', 'tech-writer', 'pm', 'ux-designer', 'architect', 'sm', 'dev', 'solo-dev']`:

    - Test 1 (slash-colon → dollar-dash): `/gm:agent-pm` → `$gm-agent-pm`
    - Test 2 (slash-colon multi-char shortName): `/gm:agent-tech-writer` → `$gm-agent-tech-writer`
    - Test 3 (bare colon → bare dash): `gm:agent-tech-writer` (CSV cell, no leading `/` or `$`) → `gm-agent-tech-writer`
    - Test 4 (negative — unknown shortName): `/gm:agent-unknown` → unchanged (not in AGENT_SOURCES)
    - Test 5 (negative — unrelated colon): `gm:something-else` → unchanged
    - Test 6 (negative — already-dash): `$gm-agent-pm` → unchanged (idempotent)
    - Test 7 (no-op for colon mode): same input → same output when `invocationSyntax === 'colon'`
    - Test 8 (longest-first match): `gm:agent-solo-dev` → `gm-agent-solo-dev` (NOT `gm-agent-solo` + `-dev`). Build the alternation with longest shortNames first.
    - Test 9 (word boundary on right): `/gm:agent-pm.` → `$gm-agent-pm.`; `/gm:agent-pmx` → unchanged (no false match on `pmx`).

    The transform helper signature:
    ```js
    function applyInvocationSyntaxTransform(text, invocationSyntax) {
      // returns text unchanged when invocationSyntax !== 'dash'
      // when 'dash', applies two substitutions over the 8 known shortNames
    }
    ```
