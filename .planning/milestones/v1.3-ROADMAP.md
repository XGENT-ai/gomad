## Milestone Summary

**Decimal Phases:** None.

**Key Decisions:**

- Marketplace workstream dropped (former Phase 10 / MARKET-01..05) — `gomad install` CLI is the single distribution path; stale `.claude-plugin/marketplace.json` removed from the repo rather than refreshed.
- 3-phase coarse structure — AGENT-* and REL-* coupled in a single phase (Phase 12) because REL-02 quality gate depends on every AGENT-* test, AGENT-10 extends `verify-tarball.js` (the release gate itself), and CHANGELOG BREAKING is specifically about the agent-dir move.
- Phase 12 last with exclusive lock — riskiest change (runtime pointer + cleanup-planner + namespace collision + latent `newInstallSet` bug); needed Phase 10 `_config/<subdir>/` pattern proven and Phase 11 docs ready for path finalization.
- DOCS-07 placed entirely in Phase 12 (not split) — `tools/validate-doc-paths.js` enforces against real post-v1.3 layout.
- BREAKING-callout pattern + `prepublishOnly = npm run quality` gate (D-07) carried forward from v1.2 BREAKING-section template; OIDC migration deferred to v1.4+.
- Agent dir relocated `<installRoot>/gomad/agents/` → `<installRoot>/_config/agents/` (agents-only; `gomad/workflows/` and `gomad/data/` stay) — centralizes user-customizable surface under `_config/`; runtime pointer + cleanup-planner + namespace collision worked through with manifest-driven backup snapshots.
- `prepublishOnly` quality matrix extended to gate on tarball / install / legacy-upgrade tests, NOT `test:orphan-refs` — the latter is a regression-ratchet handled by pre-commit; `quality` focuses on shippable-tarball correctness.
- Test install-smoke harnesses must be hermetic re: outer `npm publish` env (strip `npm_*` from spawned `npm install`) — `npm publish --registry=<custom>` exports `npm_config_registry` to subprocesses; nested `npm install` inside `prepublishOnly`-triggered tests would otherwise deadlock.
- Phase 10 `_config/<subdir>/` pattern proven via greenfield `_config/kb/` (no collision) before Phase 12 touched the collision-prone `_config/agents/` path.

**Issues Resolved:**

- `newInstallSet` derivation bug in `installer.js` (latent since v1.2) — fixed by Phase 12 / AGENT-04 via dedicated v12 branch in `cleanup-planner.js`.
- Custom-file detector treating relocated persona `.md` files as user-custom — fixed by AGENT-05 whitelist using `AgentCommandGenerator.AGENT_SOURCES`.
- Late-breaking publish-time gate fixes: orphan-refs allowlist updated for 124 Phase 11/12 `gm-agent-*` refs (`a8812fb`); prettier sweep on 3 test/installer files that bypassed pre-commit (`1a611ee`); install-smoke env hermetic strip (`df49ce9`); tarball-grep allowlist extended for shipped statusline + agent-tracker hooks (`5bec364`).
