**STACK.md section 3 BM25 algorithm (~50 LOC reference)** — described in workflow.md prose:
- Tokenize: lowercase + split on non-alphanumeric + filter tokens < 3 chars.
- Parameters: `k1 = 1.2`, `b = 0.75` (standard Okapi BM25 defaults).
- For each document: compute term frequency (tf) for each query token.
- IDF: `log(1 + (N - n + 0.5) / (n + 0.5))` where `N` is doc count and `n` is doc-frequency.
- Score: sum over query tokens of `idf(q) * (tf(q) * (k1 + 1)) / (tf(q) + k1 * (1 - b + b * (docLen / avgDocLen)))`.
- Hardcoded `NO_MATCH_FLOOR` heuristic constant (exact value planner's discretion per CONTEXT.md — recommend starting at `0.5`, tune with seeds); short comment explaining why.

**Levenshtein algorithm** — described in workflow.md prose:
- Standard edit-distance (insertion/deletion/substitution = 1 each).
- Input: target slug (user-provided) vs each directory name under `<installRoot>/_config/kb/`.
- Output: list of slugs with distance <= 3.
- Halt-not-auto-execute per D-11 (even at distance 1).

**Pack directory walk** — recursive, skipping `.`/`_`-prefixed dirs (manifest-generator.js:190-266 analog):
```
Walk <installRoot>/_config/kb/<slug>/ recursively:
  For each dir entry:
    If entry.name starts with '.' or '_': skip
    If entry is file and ends with .md: index as BM25 document
    If entry is dir: recurse
```

**D-09 listing format** (when `{query}` omitted): one line per file: `<relative_path_under_slug_root> — <H1 heading>`.
Example:
```
SKILL.md — Testing Pack Overview
reference/unit-tests.md — Unit Testing Fundamentals
reference/e2e.md — End-to-End Testing with Playwright
```
H1 heading extracted via regex on first `/^#\s+.+/m` match after frontmatter.

**D-10 "no match" threshold** — prose rule:
> Hardcoded heuristic constant `NO_MATCH_FLOOR` in workflow.md. If best BM25 score <= `NO_MATCH_FLOOR`, emit `No match for "{query}" in {slug} pack. Try gm-domain-skill {slug} with no query to list available files.`

**D-11 "did you mean" prose**:
- With suggestions: `The pack "{slug}" is not installed. Did you mean: {slug1}, {slug2}? (Based on Levenshtein distance <= 3 against installed packs.)`
- Without: `The pack "{slug}" is not installed. Installed packs: {slug_list}.`
