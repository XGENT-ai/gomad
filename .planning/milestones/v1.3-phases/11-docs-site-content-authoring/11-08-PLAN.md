    Specific edit — after the existing CUSTOM_PAGE_ROUTES skip (line ~253-255) and BEFORE the `const targetFile = resolveLink(linkPath, filePath);` line (line ~258), insert:

    ```js
    // Skip absolute URLs (http://, https://, mailto:, ftp://, file://, tel:, etc.).
    // The validator only checks site-internal markdown links; external URL targets
    // are not resolvable as local files. This prevents false positives like
    // https://github.com/xgent-ai/gomad/blob/main/README.md being chased as a local file.
    if (/^[a-z][a-z0-9+.-]*:\/\//i.test(linkPath) || linkPath.startsWith('mailto:') || linkPath.startsWith('tel:')) {
      continue;
    }
    ```

    The regex `^[a-z][a-z0-9+.-]*:\/\//i` matches any RFC-3986-shaped URL scheme followed by `://` (covers http, https, ftp, file, ws, wss, data: with `://` form, and any future scheme). The explicit `mailto:` and `tel:` checks cover schemes that don't use `://`. This is the per-D-13/CONTEXT-style "tooling fix preferred" path from 11-07-SUMMARY's two valid options for Gap A.

    Do NOT change the LINK_REGEX itself — leaving the regex permissive but adding a skip guard is safer than tightening the regex (which could accidentally skip legitimate `./..rel/path.md` links). The skip guard is the smallest-blast-radius fix.
