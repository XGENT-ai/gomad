## Directory Purposes

**bin/**
- Purpose: CLI entry point and command routing
- Contains: Single executable script using Commander framework
- Key files: `bin/mobmad-cli.js` - defines all mobmad commands (install, curate, package, sync, status, update, uninstall, mcp)

**src/module/**
- Purpose: BMAD module definition and packaged content for installation
- Contains: Module metadata, skill directories, agent definitions, registry CSV
- Key files:
  - `src/module/module.yaml` - BMAD framework configuration (module code, prompts, post-install notes)
  - `src/module/module-help.csv` - Flat registry of all packaged skills with menu codes and phase assignments
  - `src/module/agents/` - 16 BMAD-specific agents organized by category (analysis, planning, research, review)
  - `src/module/skills/` - Skill directories (generated by `mobmad package`) containing SKILL.md and bmad-skill-manifest.yaml

**src/module/agents/**
- Purpose: BMAD-specific agents for development workflow
- Contains: 16 markdown files organized in four categories
  - `bmad-analysis/` - api-documenter.md, codebase-analyzer.md, data-analyst.md, pattern-detector.md
  - `bmad-planning/` - epic-optimizer.md, requirements-analyst.md, technical-decisions-curator.md, trend-spotter.md, user-journey-mapper.md, user-researcher.md, dependency-mapper.md
  - `bmad-research/` - market-researcher.md, tech-debt-auditor.md
  - `bmad-review/` - document-reviewer.md, technical-evaluator.md, test-coverage-analyzer.md

**src/module/skills/**
- Purpose: Generated skill directories with BMAD-compatible manifests
- Contains: Subdirectories per skill (coding-standards, python-patterns, tdd-workflow, etc.)
- Pattern: Each skill has `SKILL.md` (with frontmatter) and `bmad-skill-manifest.yaml`
- Generated by: `mobmad package` command from `~/.claude/skills/`

**global/**
- Purpose: Source directory for global assets deployed to `~/.claude/`
- Contains: Pre-selected (curated) rules, hooks, agents, commands
- Used by: `mobmad install` → copies from here to `~/.claude/`
- Depends on: `mobmad sync` to populate from user's upstream `~/.claude/`

**global/rules/**
- Purpose: Coding standards, conventions, security guidelines per language
- Contains: 77 rules organized by language
  - `common/` - Universal principles (coding-style, git-workflow, testing, performance, patterns, security, hooks, agents)
  - `typescript/` - TypeScript-specific extensions
  - `python/`, `golang/`, `rust/`, `php/`, `swift/`, `java/`, `kotlin/`, `csharp/`, `cpp/`, `perl/` - Language directories
  - `zh/` - Chinese translations

**global/commands/**
- Purpose: Custom CLI commands for development workflow
- Contains: Predefined commands (e.g., plan, checkpoint, code-review, tdd, verify, docs, e2e, etc.)

**global/hooks/**
- Purpose: Automated actions before/after tool execution and at session end
- Contains: Script files (JavaScript/shell) typed as PreToolUse, PostToolUse, or Stop
- Examples: check-console-log.js (detects console.log in edited files), auto-formatters, linters

**global/agents/**
- Purpose: Global-scope agent definitions
- Contains: Agent markdown files for global context (available across all projects)

**catalog/**
- Purpose: Single source of truth for available skills, agents, commands, hooks, and preset definitions
- Contains: YAML data files with metadata and relationships
- Key files:
  - `catalog/skills.yaml` - 142 skills with name, description, category (language, framework, testing, review, security, devops, content, tool, domain, workflow), default_include boolean
  - `catalog/agents.yaml` - 48 agents with name, description, scope (global or project), default_include
  - `catalog/commands.yaml` - 70 commands with descriptions
  - `catalog/hooks.yaml` - 36 hooks with name, type (PreToolUse, PostToolUse, Stop), description
  - `catalog/presets.yaml` - 6 presets with inheritance (extend field), skill/agent/command/hook arrays, include_all boolean, additional_* fields

**tools/**
- Purpose: CLI tool implementations for curation, packaging, synchronization, and installation
- Key files:
  - `tools/curator.js` - Interactive selection (@clack/prompts) with preset resolution, writes lockfile
  - `tools/global-installer.js` - Copy assets to ~/.claude/ with backup and manifest tracking
  - `tools/sync-upstream.js` - Mirror selected content from user's ~/.claude/ to global/
  - `tools/package-skills.js` - Generate BMAD manifests, copy skills to src/module/skills/

**test/**
- Purpose: Validation of catalog integrity, preset resolution, module structure, installation flow
- Contains: 4 test files with 35 total tests using Node.js built-in test runner
- Key files:
  - `test/test-catalogs.js` - 9 tests: skills/agents/commands/hooks load, required fields, uniqueness, valid categories
  - `test/test-presets.js` - 8 tests: preset resolution, inheritance, cross-references, all items valid
  - `test/test-module.js` - 6 tests: module.yaml format, module-help.csv format, skill directories, manifests
  - `test/test-installation.js` - 7 tests: integration flow from curation through installation

## Naming Conventions

**Files:**
- CLI entry: `bin/{name}-cli.js` (e.g., `mobmad-cli.js`)
- Tools: `tools/{action}.js` (e.g., `curator.js`, `global-installer.js`)
- Tests: `test/test-{feature}.js` (e.g., `test-catalogs.js`, `test-presets.js`)
- Catalogs: `catalog/{entity}.yaml` (e.g., `skills.yaml`, `agents.yaml`)
- Config: `{name}.lock.yaml` for lockfiles, `module.yaml` for BMAD module
- Agents: Named markdown files in `src/module/agents/{category}/` (e.g., `api-documenter.md`)
- Skills: Directories under `src/module/skills/{name}/` with `SKILL.md` and manifests

**Directories:**
- Pascal case for feature areas: `bin/`, `src/`, `test/`, `tools/`, `catalog/`, `global/`
- Kebab-case for nested categories: `bmad-analysis/`, `bmad-planning/`, `full-stack` (preset name)
- Lowercase for language dirs: `python/`, `golang/`, `typescript/`, `zh/`

**Variables and Functions:**
- camelCase for functions: `loadCatalog()`, `resolvePreset()`, `backupDir()`, `collectFiles()`
- camelCase for variables: `catalogDir`, `projectRoot`, `selectedSkills`, `installManifest`
- SCREAMING_SNAKE_CASE for constants: `CATALOG_DIR`, `CLAUDE_DIR`, `PROJECT_ROOT`, `MANIFEST_PATH`

**YAML Keys:**
- Lowercase with underscores: `include_all`, `default_include`, `skill_preset`, `install_global_assets`, `additional_skills`
- Kebab-case in file names: `bmad-skill-manifest.yaml`, `mobmad-cli.js`, `module-help.csv`

## Where to Add New Code

**New Skill:**
- Add entry to `catalog/skills.yaml` with name, description, category, default_include
- Create `~/.claude/skills/{name}/` with SKILL.md and skill content (external to this repo)
- Run `mobmad sync` to copy to `global/skills/` (if curating from existing ~/.claude/)
- Run `mobmad package` to generate `src/module/skills/{name}/` with manifest

**New Agent:**
- Add entry to `catalog/agents.yaml` with name, description, scope (global or project)
- Create agent markdown file under appropriate category in `src/module/agents/{category}/{name}.md`
- Add entry to `src/module/module-help.csv` for discoverability in bmad-help

**New Command:**
- Add entry to `catalog/commands.yaml` with name and description
- Store command script in `global/commands/{name}.js` or `global/commands/{name}.sh`
- For BMAD commands: add menu code and phase to `src/module/module-help.csv`

**New Hook:**
- Add entry to `catalog/hooks.yaml` with name, type (PreToolUse, PostToolUse, Stop), description
- Create hook script in `global/hooks/{name}.js` (executed by BMAD framework)

**New Preset:**
- Add entry to `catalog/presets.yaml` under `presets:` section
- Define with skills/agents/commands/hooks arrays or use extend/include_all
- Test with `npm test` to validate cross-references

**New CLI Command:**
- Add command definition to `bin/mobmad-cli.js` using Commander:
  ```javascript
  program
    .command('my-command')
    .description('Description')
    .action(async (options) => {
      const { myFunction } = await import('../tools/my-tool.js');
      await myFunction(options);
    });
  ```
- Create implementation in `tools/my-tool.js`
- Add tests to `test/test-my-feature.js` using Node.js test runner

**New Test:**
- Add test file to `test/test-{feature}.js` using Node.js built-in test runner
- Import `{ describe, it }` from `'node:test'` and `assert` from `'node:assert/strict'`
- Test pattern: catalog loading, format validation, integration flows

## Special Directories

**bin/**
- Purpose: Executable scripts
- Generated: No
- Committed: Yes
- Pattern: Single file with `#!/usr/bin/env node` shebang

**src/module/skills/**
- Purpose: Packaged skill directories with BMAD manifests
- Generated: Yes (by `mobmad package`)
- Committed: Yes (packaged skills committed to repo)
- Pattern: Created on demand from `~/.claude/skills/` source

**global/**
- Purpose: Staging area for global assets
- Generated: Partially (populated by `mobmad sync` from ~/.claude/)
- Committed: Yes (curated content committed)
- Pattern: Source for `mobmad install` to copy to ~/.claude/

**mobmad.lock.yaml**
- Purpose: Persist user's selection from curation
- Generated: Yes (by `mobmad curate`)
- Committed: Typically yes (to share configuration across team/machines)
- Pattern: YAML with selections arrays for skills, agents, commands, hooks

**~/.claude/.mobmad-manifest.yaml (runtime)**
- Purpose: Track installed files for safe rollback
- Generated: Yes (by `mobmad install`)
- Committed: No (local installation state)
- Pattern: Manifest with asset_type sections listing files and timestamps

**~/.claude/*.mobmad-backup-* (runtime)**
- Purpose: Timestamped backups before install
- Generated: Yes (by `mobmad install` via `backupDir()`)
- Committed: No (local backups)
- Pattern: Directory copies created before overwriting existing assets
